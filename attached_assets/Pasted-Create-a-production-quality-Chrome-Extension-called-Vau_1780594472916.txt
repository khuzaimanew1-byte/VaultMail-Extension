========================================
EMAIL INPUT PROCESSING
======================

User can paste emails like:

[john@gmail.com](mailto:john@gmail.com)
[alex@gmail.com](mailto:alex@gmail.com)

or

[john@gmail.com](mailto:john@gmail.com), [alex@gmail.com](mailto:alex@gmail.com), [sam@gmail.com](mailto:sam@gmail.com)

System should:

* Split by newline
* Split by comma
* Trim spaces
* Remove duplicates
* Ignore empty values
* Store in chrome.storage.local

========================================
EMAIL CARD SYSTEM
=================

Each email should appear as an individual card.

Important:
Cards must NOT be large.

Card requirements:

* Compact height
* Single-line layout
* Premium spacing
* Modern Apple-style appearance

Each card contains:

* Email address only

No extra text.
No timestamps.
No descriptions.

Card styling:

* Rounded corners
* Subtle border
* Frosted glass effect
* Hover elevation
* Smooth appearance animation
* Text truncation if needed

Card interaction:

* Hover slightly lifts card
* Smooth shadow transition
