========================================
EVENT SYSTEM
============

Use event delegation at document level.

Use:

event.target

as the only source of truth.

Never resolve context from:

* parent elements
* container elements
* wrapper elements
* labels
* spans
* closest matching ancestors

Only the actual interacted element may create context.

If event.target itself is not a valid email input:

Return immediately.

Do nothing.

No mousedown-based capture logic.

No parent traversal for interaction detection.

========================================
STATUS DATA RETENTION
=====================

Retention is status-based.

Do not use fixed expiration for active statuses.

The currently active status must never expire regardless of how much time passes.

Status data may only become eligible for cleanup after that status is no longer active.

Example:

Processing
→ Previewing Account

When this happens:

* Processing becomes inactive.
* Start a 5 minute cleanup timer for Processing data.

If Processing becomes active again before the timer expires:

* Cancel the timer.
* Reset retention tracking.
* Continue using the existing Processing data.

The same rule applies to every temporary status.

========================================
PROCESSING STATE RETENTION
==========================

Processing data includes:

* activeContext
* emailInput
* emailValue
* activeSubmitButton
* processing timestamps

While Processing is the active status:

Never remove this data.

When Processing becomes inactive:

Start a 5 minute cleanup timer.

After 5 minutes:

Discard Processing data only if Processing has not become active again.

========================================
PREVIEW STATE RETENTION
=======================

Preview data includes:

* capturedEmail
* preview timestamps

While Previewing Account is the active status:

Never remove this data.

When Previewing Account becomes inactive:

Start a 5 minute cleanup timer.

After 5 minutes:

Discard Preview data only if Previewing Account has not become active again.

========================================
STATUS REACTIVATION RULE
========================

If a status becomes active again before its cleanup timer expires:

* Cancel pending cleanup.
* Reset the timer.
* Preserve all existing data for that status.

Only statuses that remain inactive for the full 5 minutes may be cleaned up.

========================================
SUBMIT HANDLING
===============

Capture may occur from:

* clicking the active submit button
* form submit events triggered by Enter key

Both are valid submissions.

No mousedown capture logic.

No duplicate capture paths.

A single submission must produce a single capture.
