CONTEXT ENGINE REWRITE

Refactor the extension.

Remove any logic that creates context because elements merely exist in the DOM.

Remove:

* DOM-scan based context creation
* MutationObserver-based context creation
* URL-based context creation
* Page-type based context creation
* Form-existence based context creation
* Selector-existence based context creation

Selectors are only used to validate the interacted element.

Selectors never create context by themselves.

========================================
PRIMARY RULE
============

Only user interaction creates context.

Nothing else.

If the user never interacts with a tracked email field:

No context exists.

========================================
EVENT SYSTEM
============

Use event delegation at document level.

Use:

event.target

as the source of truth.

Do not create state by scanning the DOM.

Do not create state by inspecting the page.

========================================
TRACKED EMAIL INPUTS
====================

Valid email inputs:

* input[autocomplete="email"]
* input[name="email"]
* input[name="username"]

When an event occurs:

Read event.target.

If event.target does not match a valid email input:

Return immediately.

Do nothing.

========================================
EMAIL CONTEXT CREATION
======================

Context is created only when a valid email input receives:

* input
* change

events.

Focus alone must not create context.

Context:

activeContext = {
emailInput,
emailValue,
lastUpdated
}

Update activeContext on every valid input/change event.

The latest interacted email field always becomes the active context.

If the user interacts with a different valid email field:

* Immediately discard the previous activeContext.
* Create a completely new activeContext.
* Replace all previous Processing state data.

========================================
BUTTON RESOLUTION
=================

Buttons are not tracked by ID.

Use the active email input to resolve context.

When activeContext is updated:

1. Resolve parent form using:

   * emailInput.form
   * fallback: emailInput.closest('form')

2. Inside ONLY that form find:

   * button[type="submit"]
   * input[type="submit"]

3. Store as activeSubmitButton.

A valid context requires:

* emailInput
* form
* submit button inside the same form

If validation fails:

No context is created.

========================================
BUTTON CLICK LOGIC
==================

Listen for click events.

Read:

event.target

If event.target is not the activeSubmitButton:

Return immediately.

When activeSubmitButton is clicked:

Capture:

activeContext.emailValue

Immediately save:

capturedEmail

Immediately update:

previewStatus = "Previewing Account"

Store:

capturedEmail,
lastCaptured

========================================
PROCESSING STATE
================

Processing state always represents the current activeContext.

Only one Processing context may exist.

If a newer valid email interaction occurs:

* Discard the previous Processing context.
* Replace it with the new context.
* Update lastUpdated.

The newest interaction always wins.

========================================
PREVIEW STATE
=============

After capture:

capturedEmail becomes the Preview email.

Preview always represents the most recent successful capture.

Only a newer successful capture may replace:

capturedEmail

Processing state must never overwrite Preview state.

Only a successful capture may replace Preview data.

========================================
TEMPORARY RETENTION
===================

Temporary state:

* activeContext
* Processing state
* Preview state
* capturedEmail

must survive:

* popup close
* popup reopen
* page refresh
* browser restart

Retention period:

2 hours

Store:

* data
* timestamp

If 2 hours pass without an update:

* expire the temporary state
* discard the expired context

This expiration must affect only temporary state.

========================================
PERMANENT STORAGE
=================

vaultmail_emails is permanent.

Never clear automatically.

Never remove because of:

* context changes
* page changes
* rerenders
* browser restart
* status changes
* temporary state expiration

Delete only through explicit user deletion.

========================================
SUCCESS CRITERIA
================

User types into a valid email field
↓
activeContext updates
↓
User clicks the submit button belonging to the same form
↓
Email captured
↓
Preview updated
↓
Stored emails remain intact

State is driven only by user interaction.

No DOM-driven context creation.

No URL-driven context creation.

No page-driven context creation.
